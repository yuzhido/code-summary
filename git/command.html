Git 相关操作命令 ​

一、创建于初始化 ​

bash
git init: 在当前目录初始化一个新的 Git 仓库。
git clone [url]: 克隆远程仓库到本地。
1
2

二、基本操作 ​

bash
git add . : 将文件添加到暂存区。
git commit -m “message”: 提交暂存区的文件到本地仓库。
git status: 显示工作区和暂存区的状态。
git diff: 显示文件修改的差异。
1
2
3
4

三、分支与合并 ​

bash
git branch: 列出本地分支，创建或删除分支。
git checkout [branch]: 切换到指定分支。
git merge [branch]: 合并指定分支到当前分支。
git rebase [branch]: 将当前分支变基到指定分支。
1
2
3
4

四、远程操作 ​

bash
git remote -v: 显示远程仓库的详细信息。
git fetch [remote]: 从远程仓库拉取最新变更。
git pull [remote] [branch]: 拉取远程分支并合并到本地分支。
git push [remote] [branch]: 将本地分支推送到远程仓库。
1
2
3
4

五、撤销与重置 ​

bash
git reset [file]: 从暂存区撤销文件的更改。
git revert [commit]: 撤销指定提交的更改。
git checkout – [file]: 恢复文件到最近一次提交的状态。

2

六、查看历史与日志 ​

bash
git log: 显示提交日志。
git show [commit]: 显示某次提交的详细内容。

七、标签与版本 ​

bash
git tag: 列出标签。
git tag [name]: 创建标签。
git tag -d [name]: 删除标签。

八、利用 git reflog 查看引用日志 ​

  1. git reflog 用于 1.恢复丢失的提交：在你进行合并、重置或者删除分支等操作时，可能会不小心丢失一些提交。借助 git reflog，你可以查看这些提交的引用，然后使用 git reset 或者 git cherry-pick 等命令把这些提交恢复回来。2.追踪引用的变化：它能让你清楚地了解到分支指针和 HEAD 是如何随着时间变化的，这有助于你理解和调试复杂的 Git 操作。3. 撤销错误操作：要是你执行了错误的 git reset、git checkout 或者 git rebase 等操作，可通过 git reflog 找到操作之前的引用状态，进而撤销错误操作。
bash
git reflog
1

结果如下:

bash
b6d4a83 (HEAD -> main) HEAD@{0}: commit: Add new feature
c2e8f6a HEAD@{1}: checkout: moving from old-branch to main
a1b2c3d HEAD@{2}: commit: Fix bug
1
2
3
  1. 日志的过期和清理 引用日志默认会保留一段时间（通常是 90 天），之后会被自动清理。你也可以使用 git reflog expire 命令手动清理过期的日志，使用 git gc 命令进行垃圾回收。git reflog expire 能够让你根据指定的条件来删除过期的引用日志。你可以设定日志的保留时间，从而把超出该时间范围的日志清除。 示例命令如下：
bash
# 把所有引用日志的过期时间设定为0秒，即立刻过期
git reflog expire --expire=0 --all
# 执行垃圾回收操作，实际删除过期的对象
git gc --prune=now

九、文件标识 ​

bash
A: 增加的文件.
C: 文件的一个新拷贝.
D: 删除的一个文件.
M: 文件的内容或者mode被修改了.
R: 文件名被修改了。
T: 文件的类型被修改了。
U: 文件没有被合并
X: 未知状态

十、颜色标识 ​

bash
红色，未加入版本控制;
绿色，已经加入版本控制暂未提交;
蓝色，加入版本控制，已提交，有改动；
白色，加入版本控制，已提交，无改动；
灰色：版本控制已忽略文件。
